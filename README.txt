Оконное приложение Windows Forms

Действия, если проект не запускается (не моя вина - это Windows Defender может невзлюбить мой проект). Зайдите в папку с проектом, затем выберите по очереди файлы Form1.resx, Form2.resx, Form3.resx и откройте их свойства. Затем в нижней части окна поставьте галочку в поле "Разблокировать". В любой непонятной ситуации перезапускайте Visual Studio (просто совет № 1).


Нюансы моей работы (как я понял ТЗ и как ограничил некоторые аспекты приложения):
1) Главный функционал приложения - Открыть, Редактировать и Сохранить файл, поэтому перед запуском вы обязаны выбрать его в высветившимся окне Windows (если хотите пустой текстовый файл, то можно нажать ПКМ -> "Создать" -> "Текстовый документ" и выбрать его).
2) При нажатии сохранения с выбором пути в выбранной вкладки будет открыт файл по новому пути.
3) Вы не можете открыть больше 10 вкладок в одном окне.
4) Приложение без вкладок будет закрываться.
5) Вы не можете открыть больше 5 окон приложения.
6) При закрытии основного окна все дочерние закрываются, иначе, то есть при закрытии одного из дочерних, закроется только одно окно.
7) Основной шрифт текста - это шрифт, который ставится по умолчанию для всех файлов, кроме файлов с расширениями .rtf и .cs (.csproj).
8) При перезапуске открываются ранее запущенные вкладки, избегая дублирования (чтобы 2 одинаковых по пути к файлу не было), также не открываются пустые файлы (без пути).
9) Вы не можете открывать файлы размером больше 128 мегабайт.
10) Частота сохранения файлов записывается в меню "Настройки" в секундах от 10 до 1000.
11) В меню "Настройки" вы также можете включить или выключить уведомления о автоматических сохранениях и резервном копировании (Журналировании).
12) Дополнение к форматированию текста - вы можете выбирать цвет заливки.
13) Частота создания резервных копий записывается в меню "Настройки" в минутах от 1 до 60.
14) Хранение резервных копий осуществляется по имени и времени её создания ("день. месяц. год-час.минуты.секунды.последний тик", поэтому при одинаковых именах изначальных файлов возможна путаница. Совет №3: Не пытайтесь запутать себя и программу - называйте файлы по-разному.
15) Кнопка "Журналирование" открывает резервные копии для текущего файла в вкладке.
16) Программа не будет делать резервные копии файлов, имена которых превышают 150 символов.
17) Ctrl + Z и Ctrl + Shift + Z реализованы, но и Ctrl + Y тоже работает.
18) Вы можете открывать .cs файлы (просто факт).
19) Доп не из ТЗ: Есть строка состояния, как в "Блокноте", с количеством строк, слов и символов.
20) Доп не из ТЗ: Кнопка "Время и дата" (F5) с новой строки вставляет текущую дату и время.
